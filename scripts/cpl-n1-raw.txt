Questão 1
O desenvolvimento dos primeiros compiladores está intrinsecamente ligado à evolução das
linguagens de programação de alto nível, que visavam facilitar a escrita de software em
comparação com a linguagem de montagem (Assembly). Sobre o marco histórico inicial dessa área,
qual linguagem é amplamente reconhecida por ter tido o primeiro compilador completo e funcional,
desenvolvido por John Backus na década de 1950?
A) COBOL
B) LISP
C) FORTRAN
D) ALGOL 60
E) Ada
Gabarito:
C) FORTRAN
Explicação: O FORTRAN (Formula Translation), desenvolvido pela IBM sob a liderança de John
Backus entre 1954 e 1957, é considerado o primeiro compilador de sucesso para uma linguagem
de alto nível. O projeto foi revolucionário porque provou que um compilador poderia gerar código
de máquina quase tão eficiente quanto o escrito manualmente por programadores experientes em
Assembly, superando o ceticismo da época.

Questão 2
No início da computação, a programação era feita diretamente em código de máquina ou Assembly,
o que tornava o desenvolvimento dependente da arquitetura do hardware. Com o surgimento das
linguagens de "Terceira Geração" (3GL), como o ALGOL, houve um esforço para tornar as
linguagens mais independentes de máquina. Qual inovação técnica no design de compiladores foi
amplamente impulsionada pela necessidade de descrever a sintaxe dessas novas linguagens de
forma rigorosa e sistemática?
A) A criação dos diagramas de fluxo de dados.
B) O desenvolvimento da Notação BNF (Backus-Naur Form).
C) A invenção da arquitetura de Von Neumann.
D) O uso de fitas magnéticas para armazenamento de símbolos.
E) A implementação exclusiva de interpretadores em vez de compiladores.
Gabarito:
B) O desenvolvimento da Notação BNF (Backus-Naur Form).
Explicação: A Notação BNF foi introduzida por John Backus (e refinada por Peter Naur) para
descrever a sintaxe da linguagem ALGOL 60. Essa técnica permitiu uma definição formal e
matemática das gramáticas das linguagens, facilitando imensamente a construção de analisadores
sintáticos (parsers) automáticos e robustos, sendo utilizada até hoje na definição de quase todas
as linguagens modernas.

Questão 3
Durante a década de 1950, Grace Hopper e sua equipe desenvolveram o sistema A-0, que é
frequentemente citado na história da computação. Embora operasse de forma diferente dos
compiladores modernos de otimização, qual foi a principal contribuição conceitual desse sistema
para a área de tradução de linguagens?
A) A invenção da técnica de interpretação linha a linha em tempo real.
B) A criação da primeira linguagem de programação orientada a objetos.
C) O conceito de usar sub-rotinas pré-compiladas que poderiam ser combinadas para formar um
programa completo.
D) A implementação da primeira interface gráfica para depuração de código de máquina.
E) O desenvolvimento do primeiro sistema operacional capaz de multitarefa.
Gabarito:
C) O conceito de usar sub-rotinas pré-compiladas que poderiam ser combinadas para formar um
programa completo.
Explicação: O sistema A-0 (Arithmetic Language version 0) funcionava mais como um "carregador"
ou "ligador" (linker) primitivo. Ele permitia que programadores usassem códigos mnemônicos para
chamar bibliotecas de sub-rotinas matemáticas já existentes, "compilando" (reunindo) essas partes
em um programa executável. Foi um passo fundamental para distanciar o programador do
hardware e introduzir a ideia de reutilização de código.

Questão 04
No contexto da estrutura das linguagens de programação, os elementos são geralmente
classificados em diferentes níveis de abstração. O nível que define as regras de formação de frases
válidas (como a estrutura de um comando if ou a declaração de uma função), determinando como
os tokens devem ser agrupados, é chamado de:
A) Sintaxe.
B) Semântica Estática.
C) Semântica Dinâmica.
D) Léxico.
E) Pragmática.
Gabarito:
A) Sintaxe.
Explicação: A sintaxe é o conjunto de regras que define a forma e a estrutura das construções
permitidas em uma linguagem (a "gramática"). Enquanto o léxico lida com as palavras individuais
(tokens), a sintaxe determina se a combinação dessas palavras forma uma instrução válida para o
compilador.

Questão 05
Um programador escreve a seguinte linha de código em uma linguagem fictícia: int x = "vinte";. O
compilador identifica que, embora a construção do comando esteja gramaticalmente correta (segue
a ordem tipo-identificador-atribuição-valor), há uma incompatibilidade de tipos, pois uma variável
inteira não pode receber uma string. Em qual nível de análise da estrutura da linguagem essa falha
é detectada?
A) Análise Léxica.
B) Análise Sintática.
C) Semântica Estática.
D) Pragmática.
E) Geração de Código Intermediário.
Gabarito:
C) Semântica Estática.
Explicação: A semântica lida com o "significado" e a validade lógica das construções. A verificação
de tipos (type checking) em tempo de compilação é chamada de semântica estática. O analisador
sintático confirma que a frase está bem formada, mas o analisador semântico é quem verifica se
os tipos de dados envolvidos fazem sentido para a operação pretendida.

Questão 06
Dentro do estudo da estrutura das linguagens de programação, existe uma dimensão que se
preocupa com a relação entre os símbolos da linguagem, seus significados e o contexto de uso
pelos programadores e pelo sistema (como eficiência, legibilidade e intenção). Essa dimensão, que
lida com o "uso prático" e a interpretação humana ou do ambiente de execução, é denominada:
A) Ortogonalidade.
B) Pragmática.
C) Gramática Regular.

Gabarito:

escrita, a performance esperada de certas construções no hardware e como o programador
expressa suas intenções através das ferramentas disponíveis na linguagem.

Questão 07
Os paradigmas de programação definem a forma como o programador estrutura sua solução e
como o computador a executa. Um paradigma específico se baseia no conceito de "efeitos
colaterais" e na alteração de estados por meio de comandos sequenciais que modificam valores
na memória. Qual é o nome desse paradigma, que é a base de linguagens como C e Pascal?
A) Paradigma Funcional.
B) Paradigma Declarativo.
C) Paradigma Imperativo.
D) Paradigma Lógico.
E) Paradigma Orientado a Eventos.
Gabarito:
C) Paradigma Imperativo.
Explicação: O paradigma imperativo foca no "como" a tarefa deve ser realizada, utilizando
comandos que alteram o estado do programa (variáveis na memória) de forma sequencial. É o
modelo mais próximo da arquitetura de Von Neumann, onde o compilador traduz instruções que
manipulam diretamente os registros e a memória da máquina.

Questão 08
Diferente do paradigma imperativo, que foca no passo a passo de "como" realizar uma tarefa, existe
um paradigma onde o programador descreve "o que" deve ser alcançado (o resultado desejado ou
as propriedades lógicas), deixando para o sistema (compilador ou interpretador) a responsabilidade
de determinar os passos de execução. Esse paradigma é conhecido como:
A) Paradigma Estruturado.
B) Paradigma Declarativo.
C) Paradigma Procedural.
D) Paradigma Orientado a Objetos.
E) Paradigma de Baixo Nível.
Gabarito:
B) Paradigma Declarativo.
Explicação: No paradigma declarativo (que engloba o funcional e o lógico), o foco está na
especificação do problema e das relações entre os dados, em vez de uma sequência de comandos
que alteram estados. Linguagens como SQL, Prolog e Haskell são exemplos onde o "o quê" é mais
importante do que o fluxo de controle detalhado.

Questão 09
Um dos paradigmas mais utilizados na engenharia de software moderna baseia-se no agrupamento
de dados e dos comportamentos que os manipulam em unidades coesas. Esse modelo utiliza
conceitos como herança, polimorfismo e encapsulamento para facilitar a reutilização de código e a
manutenção de sistemas complexos. Qual é este paradigma?
A) Paradigma de Script.
B) Paradigma Orientado a Objetos.
C) Paradigma de Fluxo de Dados.
D) Paradigma de Marcação.
E) Paradigma de Baixo Nível.
Gabarito:
B) Paradigma Orientado a Objetos.

isso implica em gerenciar tabelas de métodos virtuais, hierarquias de classes e o tempo de vida de
instâncias na memória (geralmente no heap).

Questão 10
Um compilador é um programa de sistema que traduz um código escrito em uma linguagem de alto
nível (linguagem fonte) para uma linguagem de baixo nível (linguagem alvo), como código de
máquina ou assembly. Qual é a característica fundamental que define o processo de compilação
pura, diferenciando-o da interpretação?
A) A execução do programa ocorre simultaneamente à tradução de cada linha de código.
B) O compilador gera um arquivo executável independente, que pode ser rodado sem a
necessidade do código fonte original.
C) O processo de compilação dispensa a necessidade de análise sintática e léxica do código.
D) Compiladores são utilizados exclusivamente para linguagens de marcação, como HTML e XML.
E) A tradução é realizada apenas no momento em que o usuário final interage com a interface do
programa.
Gabarito:
B) O compilador gera um arquivo executável independente, que pode ser rodado sem a
necessidade do código fonte original.
Explicação: Diferente do interpretador, que traduz e executa o código "on-the-fly" (em tempo de
execução), o compilador realiza a tradução completa de uma só vez, gerando um objeto ou arquivo
binário. Uma vez gerado esse executável, ele pode ser distribuído e rodado em máquinas
compatíveis sem que o código original ou o próprio compilador precisem estar presentes.

Questão 11
Durante o processo de tradução, alguns compiladores realizam todo o trabalho de análise e
geração de código em uma única leitura do código-fonte, enquanto outros dividem o trabalho em
várias etapas, percorrendo representações do programa múltiplas vezes. Como é denominado o
modelo de compilador que processa o código de uma só vez, do início ao fim, para gerar o código
alvo?
A) Compilador Multi-pass (múltiplos passos).
B) Compilador Cruzado (Cross-compiler).
C) Compilador One-pass (passo único).
D) Decompilador.
E) JIT (Just-In-Time).
Gabarito:
C) Compilador One-pass (passo único).
Explicação: Um compilador one-pass é projetado para ler o programa fonte e gerar o código
equivalente em uma única passagem. Embora sejam extremamente rápidos, esses compiladores
geralmente têm limitações na capacidade de otimização e na resolução de referências "para frente"
(como usar uma variável antes de sua declaração), comparados aos compiladores de múltiplos
passos que analisam o código mais profundamente.

Questão 12
No desenvolvimento de software, é comum que o computador onde o código é escrito e compilado
(máquina hospedeira) possua uma arquitetura diferente do computador onde o programa será
efetivamente executado (máquina alvo). Um compilador que é executado num sistema (ex:
Windows/Intel), mas gera código executável para um sistema diferente (ex: Android/ARM), é
conhecido como:
A) Decompilador.
B) Compilador Cruzado (Cross-compiler).
C) Pré-processador.
) Interpretador Híbrido.
E) Compilador JIT (Just-in-Time).
Gabarito:
B) Compilador Cruzado (Cross-compiler).
Explicação: Um cross-compiler é uma ferramenta fundamental para o desenvolvimento em
plataformas que não possuem recursos de hardware suficientes para rodar um compilador próprio
(como microcontroladores) ou para criar sistemas operativos para novas arquiteturas. Ele permite
que o desenvolvedor utilize o poder de processamento de uma estação de trabalho para gerar
binários para o hardware alvo.

Questão 13
Embora tanto compiladores quanto interpretadores tenham o objetivo de permitir a execução de
programas escritos em linguagens de alto nível, eles utilizam estratégias distintas. Uma vantagem
característica do modelo de interpretação em relação à compilação pura é:
A) Maior velocidade de execução do programa final, pois o código de máquina é gerado
previamente.
B) Facilidade na depuração (debugging) e maior portabilidade do código-fonte entre diferentes
plataformas.
C) Menor consumo de memória RAM durante a execução do programa.
D) Garantia de que todos os erros de lógica serão detectados antes de o programa iniciar a
execução.
E) Produção de um arquivo binário (.exe ou similar) que protege o código-fonte original.
Gabarito:
B) Facilidade na depuração (debugging) e maior portabilidade do código-fonte entre diferentes
plataformas.
Explicação: Como o interpretador traduz o código instrução por instrução em tempo real, ele pode
interromper a execução exatamente no ponto onde ocorreu um erro, facilitando a identificação de
falhas. Além disso, o mesmo código-fonte pode rodar em qualquer sistema que possua o
interpretador instalado, sem a necessidade de gerar um binário específico para cada arquitetura de
processador.

Questão 14
Linguagens como Java e C# utilizam um modelo de tradução intermediário para equilibrar
desempenho e portabilidade. Nesse modelo, o código-fonte é primeiro traduzido para uma
linguagem de baixo nível independente de máquina (como o Bytecode ou CIL), que é então
executada por uma máquina virtual. Qual componente de software é responsável por traduzir esse
código intermediário em código de máquina nativo durante a execução, visando aumentar a
performance?
A) Compilador Cruzado (Cross-compiler).
B) Pré-processador.
C) Compilador JIT (Just-In-Time).
D) Montador (Assembler).
E) Descompilador.
Gabarito:
C) Compilador JIT (Just-In-Time).
Explicação: O compilador JIT faz parte da máquina virtual. Em vez de interpretar o bytecode
instrução por instrução (o que seria lento), o JIT compila partes do código (geralmente as mais
executadas, chamadas de "hot spots") diretamente para o código de máquina do processador real
no momento da execução. Isso permite que o programa rode com uma velocidade próxima à de
um código nativo, mantendo a portabilidade.

Gabarito:
C) Otimizador de Consultas (Query Optimizer).
Explicação: Assim como um compilador otimiza o código para que ele rode mais rápido no
